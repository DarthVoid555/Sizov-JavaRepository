Что такое Pull Request (PR):
PR - это запрос на внесение изменений в основную ветку проекта.
Он позволяет другим участникам проекта (или преподавателю) проверить код, обсудить его и принять изменения.

Зачем нужен PR:
Позволяет вести контроль качества кода
Делает работу отслеживаемой и прозрачной
Интегрируется с CI/CD и статическим анализом


Как работает процесс:

1 Создание новой ветки для выполнения задания

2 Написание кода и коммит изменения в эту ветку

3 Делаем Push на удалённый репозиторий

4 При создании PR, далее автоматически запускается CI и анализатор кода

5 Проверяем результаты анализа и исправляем ошибки

6 После одобрения PR - изменения вливаются в основную ветку


Автоматическая сборка проекта (CI):
В репозитории настроен GitHub Actions, который запускает сборку проекта и анализ кода при push и PR.
Это гарантирует, что проект собирается корректно и код соответствует стандартам качества.

GitVerse / GitHub
Для работы с репозиторием используется GitVerse или GitHub
Создаём учетную запись, если её ещё нет
Клонируем репозиторий и работаем в отдельных ветках

Статические анализаторы кода:
Подключен SonarCloud
Анализирует качество кода, ищет ошибки, баги и code smells
Запуск происходит автоматически при push или PR
Помогает поддерживать чистоту и читаемость кода
